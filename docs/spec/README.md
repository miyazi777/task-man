# task-man 仕様書

task-man の仕様を機能領域ごとにまとめたドキュメントです。リポジトリ全体 (Go 実装、永続化フォーマット、TUI、CLI) を解析した結果から作成しています。

このディレクトリのドキュメントはリポジトリ実装 (`cmd/`, `internal/`) を一次ソースとして記述されています。実装と齟齬が生じた場合はコードを優先し、本仕様書を追従更新してください。

## 用語

- **タスク (task)**: 作業項目。タイトル・ステータス・タグ・拡張項目値・添付ファイルを持つ。サブタスクで木構造を構成する。
- **ステータス (status)**: タスクの状態。並び順 (sequence) と表示色を持ち、ユーザー定義可能。デフォルトは `todo` / `doing` / `done` の 3 種類。
- **拡張項目 (field)**: 任意のメタデータ列。`text` / `date` / `url` の 3 種類の型を持つ。
- **タグ (tag)**: タスクに付与する短いラベル。1 タスクあたり最大 5 個。
- **タスクデータディレクトリ**: タスクごとに割り当てられる `task-<id>/` ディレクトリ。`memo.md` を含む添付ファイル群を格納する。
- **ゴミ箱 (trash)**: `is_trash_box: true` を立てたタスクの論理削除状態。元のステータス / 階層は保持される。
- **オープナー (file opener)**: 拡張子ごとに外部アプリケーションを起動する仕組み。
